Grocery cart-fill

An MCP server that fills a Whole Foods / Amazon cart from a structured grocery list — for your review, never submitted.

This app is driven over MCP

There is no live cart-fill button here. The cart fill runs as a host-side browser automation; this app's tools surface on /grocery-cart/mcp:

  1. fill_cart(grocery_list) — request a cart fill ([{ item, quantity, preferences? }]) and get back a request_id handle.
  2. cart_fill_status(request_id) — poll for progress and the result (added / not_added / cart_url).

Never-checkout rail

The cart is filled for your review and the order-submit path is denied at the egress boundary — the automation can never place an order. You confirm checkout yourself.
